Level 1 Foundation Concepts (Prerequisites)

Learn these first - they're building blocks for everything else

1.1 Core Architecture
Business Problem

Developers need a clear understanding of LangChain's structure before building applications. Without knowing the architecture, it's difficult to choose the right components and integrate them effectively.

LangChain Solution

LangChain provides a modular architecture with separate packages for different functionalities, allowing developers to use only what they need.

Installation & Setup:
pip install langchain-core langchain langchain-community langchain-openai

Package Overview:
# langchain-core: Base abstractions and interfaces
# langchain: High-level components and chains
# langchain-community: Third-party integrations
# langchain-openai: OpenAI-specific integrations

Architecture & Ecosystem:
# LangChain is composed of several packages:
# - Core: Base interfaces and abstractions
# - Partners: Integrations with specific providers
# - Community: Community-maintained integrations
# - Text: Text-specific utilities
# - Experimental: Experimental features

1.3 Fundamental Interfaces
Business Problem

Developers need consistent ways to interact with different LangChain components, regardless of their specific implementation details.

LangChain Solution

The Runnable interface provides a standard way to execute components with methods like invoke, stream, and batch.

Runnable Interface:
from langchain.schema.runnable import Runnable

# All components implement the Runnable interface
result = component.invoke({"input": "Hello"})

# Streaming output
for chunk in component.stream({"input": "Hello"}):
    print(chunk)

# Batch processing
results = component.batch([{"input": "Hello"}, {"input": "Hi"}])

Input/Output Types:
# Components have defined input and output types
from langchain.schema.runnable import RunnablePassthrough

# Pass through input unchanged
passthrough = RunnablePassthrough()
result = passthrough.invoke({"key": "value"}) # Returns {"key": "value"}

Async Programming Basics:
# Async versions of methods are available
result = await component.ainvoke({"input": "Hello"})

async for chunk in component.astream({"input": "Hello"}):
    print(chunk)

results = await component.abatch([{"input": "Hello"}, {"input": "Hi"}])

2.1 Language Model Types
Business Problem

Applications need different types of language models for different tasks - some require conversational abilities, while others need simple text completion.

LangChain Solution

LangChain supports both modern chat models (message-based) and legacy LLMs (string-based) with unified interfaces.

Chat Models:
from langchain.chat_models import ChatOpenAI, ChatAnthropic
from langchain.schema import HumanMessage, AIMessage, SystemMessage

# Initialize chat model
chat_model = ChatOpenAI(model="gpt-3.5-turbo")
# or
chat_model = ChatAnthropic(model="claude-3-opus-20240229")

# Invoke with message objects
response = chat_model.invoke([
    SystemMessage(content="You are a helpful assistant"),
    HumanMessage(content="Hello")
])

LLMs (Legacy):
from langchain.llms import OpenAI, HuggingFaceHub

# Initialize LLM
llm = OpenAI(model="gpt-3.5-turbo-instruct")
# or
llm = HuggingFaceHub(repo_id="google/flan-t5-large")

# Invoke with string
response = llm.invoke("Hello")

Model Selection & Integration:
# Choose the right model for your use case
# - Chat models for conversational applications
# - LLMs for simple text completion
# - Consider cost, speed, and quality requirements

# Switch between models easily
from langchain.chat_models import ChatOpenAI, ChatAnthropic

def get_model(provider="openai"):
    if provider == "openai":
        return ChatOpenAI()
    elif provider == "anthropic":
        return ChatAnthropic()

2.2 Model Operations
Business Problem

Applications need efficient ways to interact with language models, especially when processing multiple requests or streaming responses.

LangChain Solution

LangChain provides standardized methods for invocation, streaming, batching, and caching model responses.

Basic Invocation:
response = model.invoke("Hello")
# or for chat models
response = model.invoke([HumanMessage("Hello")])

Streaming:
# Stream responses in real-time
for chunk in model.stream("Tell me a story"):
    print(chunk.content, end="")

# Async streaming
async for chunk in model.astream("Tell me a story"):
    print(chunk.content, end="")

Batch Processing:
# Process multiple inputs efficiently
responses = model.batch(["Hello", "Hi", "Hey"])

# Async batch processing
responses = await model.abatch(["Hello", "Hi", "Hey"])

Caching Model Responses:
from langchain.cache import InMemoryCache, SQLiteCache

# In-memory cache
langchain.llm_cache = InMemoryCache()

# Persistent cache
langchain.llm_cache = SQLiteCache(database_path=".langchain.db")

# Subsequent identical calls will use cache
response1 = model.invoke("What is AI?")
response2 = model.invoke("What is AI?") # Returns cached result

Token Usage Tracking:
# Track token usage for cost monitoring
from langchain.callbacks import get_openai_callback

with get_openai_callback() as cb:
    response = model.invoke("Tell me a long story")
    print(f"Total Tokens: {cb.total_tokens}")
    print(f"Total Cost: ${cb.total_cost}")

Rate Limiting:
# Handle rate limiting gracefully
from langchain.rate_limiting import BaseRateLimiter

class CustomRateLimiter(BaseRateLimiter):
    def __init__(self, requests_per_minute=60):
        self.requests_per_minute = requests_per_minute
        self.last_request_time = 0

    def acquire(self):
        # Implement rate limiting logic
        pass

2.3 Model Configuration
Business Problem

Applications need to fine-tune model behavior for specific use cases, controlling creativity, response length, and other parameters.

LangChain Solution

LangChain provides standardized parameters and runtime configuration options for controlling model behavior.

Standard Parameters:
from langchain.chat_models import ChatOpenAI

# Configure model with standard parameters
model = ChatOpenAI(
    temperature=0.7, # Controls randomness (0-1)
    max_tokens=500, # Limits response length
    top_p=0.9, # Nucleus sampling (0-1)
    frequency_penalty=0, # Penalize frequent tokens
    presence_penalty=0, # Penalize new tokens
    model_name="gpt-3.5-turbo",
    openai_api_key="your-api-key"
)

Runtime Configuration:
from langchain.schema.runnable import RunnableConfig

# Create runtime configuration
config = RunnableConfig(
    tags=["production"],
    metadata={"user_id": "123"},
    max_concurrency=5
)

# Use configuration with model invocation
response = model.invoke("Hello", config=config)

Local vs Remote Models:
# Using remote models (API-based)
openai_model = ChatOpenAI()
anthropic_model = ChatAnthropic()

# Using local models
from langchain.llms import LlamaCpp

local_model = LlamaCpp(
    model_path="./models/llama-2-7b-chat.ggmlv3.q4_0.bin",
    temperature=0.75,
    n_ctx=2048
)

# Using Hugging Face models locally
from langchain.llms import HuggingFacePipeline
from transformers import AutoModelForCausalLM, AutoTokenizer, pipeline

model_id = "gpt2"
tokenizer = AutoTokenizer.from_pretrained(model_id)
model = AutoModelForCausalLM.from_pretrained(model_id)
pipe = pipeline("text-generation", model=model, tokenizer=tokenizer)
hf_model = HuggingFacePipeline(pipeline=pipe)

3.1 Prompt Engineering
Business Problem

Applications need effective ways to structure prompts to get desired outputs from language models, including examples and specific formatting.

LangChain Solution

LangChain provides prompt templates, few-shot prompting, and example selectors for effective prompt engineering.

Prompt Templates:
from langchain.prompts import PromptTemplate, ChatPromptTemplate

# Basic prompt template
template = "Explain {topic} in simple terms"
prompt = PromptTemplate(template=template, input_variables=["topic"])
formatted = prompt.format(topic="quantum computing")

# Chat prompt template
chat_prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant that explains {topic}"),
    ("human", "Explain {topic} in simple terms")
])
formatted_messages = chat_prompt.format_messages(topic="quantum computing")

Few-Shot Prompting:
from langchain.prompts import FewShotPromptTemplate, PromptTemplate

# Define examples
examples = [
    {"question": "What is AI?", "answer": "Artificial Intelligence"},
    {"question": "What is ML?", "answer": "Machine Learning"}
]

# Create example prompt template
example_prompt = PromptTemplate(
    input_variables=["question", "answer"],
    template="Question: {question}\nAnswer: {answer}"
)

# Create few-shot prompt template
few_shot_prompt = FewShotPromptTemplate(
    examples=examples,
    example_prompt=example_prompt,
    suffix="Question: {input}",
    input_variables=["input"]
)

# Format with new input
formatted = few_shot_prompt.format(input="What is DL?")

Example Selectors:
from langchain.prompts.example_selector import (
    LengthBasedExampleSelector,
    SemanticSimilarityExampleSelector,
    MaxMarginalRelevanceExampleSelector
)
from langchain.vectorstores import FAISS
from langchain.embeddings import OpenAIEmbeddings

# Length-based selector (chooses examples based on input length)
length_selector = LengthBasedExampleSelector(
    examples=examples,
    example_prompt=example_prompt,
    max_length=100
)

# Semantic similarity selector (chooses similar examples)
embeddings = OpenAIEmbeddings()
vectorstore = FAISS.from_texts(
    [e["question"] for e in examples], embeddings
)
semantic_selector = SemanticSimilarityExampleSelector(
    vectorstore=vectorstore,
    k=2
)

# Max marginal relevance selector (balances similarity and diversity)
mmr_selector = MaxMarginalRelevanceExampleSelector(
    vectorstore=vectorstore,
    k=2
)

3.2 Output Processing
Business Problem

Applications need to parse and structure model outputs for use in downstream systems, ensuring consistent and reliable data extraction.

LangChain Solution

LangChain provides output parsers and structured output methods to transform model responses into structured data.

Output Parsers:
from langchain.output_parsers import (
    PydanticOutputParser,
    XMLOutputParser,
    YAMLOutputParser,
    CommaSeparatedListOutputParser,
    StructuredOutputParser,
    OutputFixingParser
)
from pydantic import BaseModel, Field

# Pydantic output parser
class Response(BaseModel):
    answer: str = Field(description="Answer to the question")
    sources: List[str] = Field(description="Sources used to answer")

parser = PydanticOutputParser(pydantic_object=Response)
prompt = PromptTemplate(
    template="Answer the question.\n{format_instructions}\nQuestion: {question}",
    input_variables=["question"],
    partial_variables={"format_instructions": parser.get_format_instructions()}
)

# Parse the model output
formatted_prompt = prompt.format(question="What is AI?")
model_output = model.invoke(formatted_prompt)
parsed = parser.parse(model_output)

Tool Calling:
from langchain.tools import tool
from langchain.chat_models import ChatOpenAI

# Define a tool
@tool
def get_weather(location: str) -> str:
    """Get weather for a location."""
    # In a real app, this would call a weather API
    return f"Weather in {location}: 72Â°F and sunny"

# Bind tools to model
model = ChatOpenAI(model="gpt-4")
model_with_tools = model.bind_tools([get_weather])

# Model will decide to call the tool
response = model_with_tools.invoke("What's the weather in Boston?")
# Response will include tool_calls if the model decides to use the tool

4.2 Data Processing
Business Problem

Large documents need to be split into smaller chunks for processing, while preserving semantic meaning and context.

LangChain Solution

LangChain provides text splitters that divide documents into meaningful chunks while preserving context.

Text Splitters:
from langchain.text_splitter import (
    RecursiveCharacterTextSplitter,
    CharacterTextSplitter,
    SemanticChunker,
    TokenTextSplitter,
    HTMLHeaderTextSplitter,
    MarkdownHeaderTextSplitter,
    RecursiveJsonSplitter,
    Language,
    PythonCodeTextSplitter
)

# Recursive splitting (preserves paragraphs, sentences)
recursive_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200
)
chunks = recursive_splitter.split_documents(documents)

# Character-based splitting
char_splitter = CharacterTextSplitter(
    separator="\n",
    chunk_size=1000,
    chunk_overlap=200
)
char_chunks = char_splitter.split_documents(documents)

# Semantic splitting (based on content meaning)
from langchain.embeddings import OpenAIEmbeddings

embeddings = OpenAIEmbeddings()
semantic_splitter = SemanticChunker(embeddings=embeddings)
semantic_chunks = semantic_splitter.split_text(long_text)

from langchain.embeddings import (

# OpenAI embeddings
openai_embeddings = OpenAIEmbeddings()
text_embedding = openai_embeddings.embed_query("Hello world")
doc_embeddings = openai_embeddings.embed_documents(["Hello", "World"])

# Hugging Face embeddings
